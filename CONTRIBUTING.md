
3. **Create a feature branch**
```bash
git checkout -b feature/your-feature-name
```

4. **Make your changes**
- Write clean, readable code
- Add comments for complex logic
- Follow existing code style
- Test your changes

5. **Commit with clear messages**
```bash
git add .
git commit -m "Add: brief description of your changes"
```

6. **Push and create Pull Request**
```bash
git push origin feature/your-feature-name
```

## ğŸ“ Code Style Guidelines

### Python
- Follow PEP 8 style guide
- Use meaningful variable names
- Add docstrings to functions
- Keep functions focused and small

### Jupyter Notebooks
- Clear markdown explanations before code
- Use heading hierarchy properly
- Include output for important cells
- Test all cells before committing

### Commit Messages
Format: `Type: Brief description`

Types:
- `Add`: New feature or content
- `Fix`: Bug fix
- `Update`: Modify existing content
- `Docs`: Documentation changes
- `Refactor`: Code restructuring
- `Test`: Add or modify tests

Examples:
```
Add: Temperature experiment in Chapter 2
Fix: Token counting error in RAG example
Update: Improve explanation in prompting basics
Docs: Add setup instructions for Windows
```
